# Milestone 1.5 Design

**Issues:** #21 (duplicate detection), #78 (age cutoff), #94 (per-feed counts), #96 (mute words), #97 (spoiler/warning tags)
**Branch:** `release/milestone1.5`
**Date:** 2026-06-14

---

## Overview

Five feed features implemented via a hybrid server/client architecture: data concerns (age cutoff, mute words, dedup, per-account limits) are handled server-side in `FeedAggregator`; presentation concerns (blur vs reveal for sensitive content) are handled client-side in React.

---

## 1. Data Model

### Migration 1 ‚Äî `users.feed_preferences` (JSON, nullable)

```json
{
  "mute_words": ["word1", "word2"],
  "max_age_days": 7,
  "cw_behavior": "blur",
  "sensitive_media_behavior": "blur"
}
```

- `cw_behavior` / `sensitive_media_behavior`: `"skip" | "blur" | "show"`, default `"blur"`
- `max_age_days`: integer or `null` (no cutoff), default `7`
- `mute_words`: string array, default `[]`

### Migration 2 ‚Äî `social_accounts.feed_settings` (JSON, nullable)

```json
{
  "max_posts": 20,
  "max_age_days": null
}
```

- `max_posts`: overrides global `per_provider_limit` for this account; default `20`
- `max_age_days`: per-account override; if set, takes precedence over user-level `max_age_days`; `null` means inherit

### `HasJsonPreferences` trait

A shared trait used by both `User` (column: `feed_preferences`) and `SocialAccount` (column: `feed_settings`). Provides:

- `getPreference(string $key, mixed $default = null): mixed`
- `setPreference(string $key, mixed $value): void`
- `getPreferences(): array` ‚Äî full merged array (stored values over defaults)

Each model declares its column name and default values; the trait handles JSON decode/encode and merging.

## 2. PostNormalizer Changes

Two new fields added to every normalised post (both Mastodon and Bluesky):

### `cw_text: string|null`

The author's content warning text, if any.

- **Mastodon:** `$source['spoiler_text']` ‚Äî use as-is if non-empty, otherwise `null`
- **Bluesky:** Derive from `$post['labels']` array of `{val: string}`:
  - `sexual`, `nudity`, `porn` ‚Üí `"Adult content"`
  - `graphic-media`, `gore` ‚Üí `"Graphic media"`
  - Other label vals ‚Üí use val as-is
  - No labels ‚Üí `null`

### `sensitive_media: bool`

Whether media is specifically flagged as sensitive.

- **Mastodon:** `$source['sensitive']` boolean
- **Bluesky:** `true` if any label val is in `[sexual, nudity, graphic-media, porn, gore]`; otherwise `false`

The normaliser remains purely a data mapper ‚Äî no filtering or behavioral logic here.

## 3. FeedAggregator Changes

Filtering pipeline applied within `fetch()`, in this order per-account then globally:

### Per-account limits (#94)

Replace `config('feed.per_provider_limit', 20)` with:
```php
$account->getPreference('max_posts', config('feed.per_provider_limit', 20))
```

### Age cutoff (#78)

After normalisation, filter each account's posts. A post is kept if:
- `boosted_by` is not null (boosts always show regardless of age), **or**
- `created_at >= now() - $maxAgeDays`

`$maxAgeDays` resolves as: account-level `max_age_days` ‚Üí user-level `max_age_days` ‚Üí `config('feed.max_age_days', 7)`. If resolved to `null`, no cutoff is applied.

### Cross-platform dedup (#21)

After the existing URL dedup pass, a second content similarity pass:

1. Normalise body: lowercase, strip punctuation, collapse whitespace, remove hashtags and URLs
2. Skip posts with normalised body length < 30 chars (too short to deduplicate reliably)
3. For each post, compare against all already-seen normalised bodies using `similar_text()`
4. Drop the post if similarity ‚â• 80% **and** posts are within 24h of each other

### Mute words (#96)

After dedup, filter posts where `body` contains any mute word (case-insensitive). Uses `$user->getPreference('mute_words', [])`. If the list is empty, skip the check entirely.

### Full pipeline order

```
per account:
  fetch from provider ‚Üí normalise ‚Üí age filter
merge all accounts ‚Üí sort by created_at desc ‚Üí URL dedup ‚Üí content dedup ‚Üí mute words ‚Üí take(buffer_size)
```

## 4. Frontend ‚Äî Spoiler/Warning Handling (#97)

### Type changes (`resources/js/types/post.ts`)

```ts
cw_text: string | null;
sensitive_media: boolean;
```

### Props from `FeedController`

```ts
cwBehavior: 'skip' | 'blur' | 'show';
sensitiveMediaBehavior: 'skip' | 'blur' | 'show';
```

Read from `$user->getPreference('cw_behavior', 'blur')` and `$user->getPreference('sensitive_media_behavior', 'blur')`.

### Skip behaviour

Handled in `useFeedQueue` ‚Äî posts filtered out before enqueuing, so the feed page never sees them. A post is skipped if:
- `cw_text !== null && cwBehavior === 'skip'`, **or**
- `sensitive_media && sensitiveMediaBehavior === 'skip'`

### Blur behaviour

Two independent overlays in `PostContent`:

**CW overlay** ‚Äî covers the post body (not media):
- Shows `cw_text` as the message
- "Show anyway" button reveals body inline, per-post, not persisted

**Sensitive media overlay** ‚Äî covers the media area only:
- Body text remains visible
- "Show anyway" button reveals media inline, per-post, not persisted
- If post has no media, overlay is not shown even if `sensitive_media` is true

### Show behaviour

`cw_text` and `sensitive_media` fields are ignored in rendering.

## 5. Settings UI

### New page: `/settings/feed`

Added to existing settings layout. Saved via `PUT /settings/feed` ‚Üí `FeedSettingsController@update`.

**Sections:**

- **Age cutoff** ‚Äî number input + "days" label; "No limit" checkbox sets value to `null`. Maps to `max_age_days`.
- **Mute words** ‚Äî tag-style input: type word/phrase, Enter to add, √ó to remove. Maps to `mute_words`.
- **Content warnings** ‚Äî select: `Show` / `Blur (tap to reveal)` / `Skip`. Maps to `cw_behavior`.
- **Sensitive media** ‚Äî select: `Show` / `Blur (tap to reveal)` / `Skip`. Maps to `sensitive_media_behavior`.

### Per-account settings on `/settings/connections`

Each connected account card gets an expandable "Feed settings" inline section:

- **Max posts** ‚Äî number input. Maps to `social_accounts.feed_settings.max_posts`.
- **Age cutoff override** ‚Äî number input + "No limit" checkbox; empty inherits user-level setting. Maps to `social_accounts.feed_settings.max_age_days`.

Saved via `PUT /settings/connections/{account}/feed` ‚Üí `FeedSettingsController@updateAccount`.

Both endpoints return Inertia redirects back with a flash status message.

## Open Decisions Resolved

| Question | Decision |
|----------|----------|
| User prefs storage | JSON column on `users` (`feed_preferences`) |
| Per-feed counts storage | JSON column on `social_accounts` (`feed_settings`) |
| Shared JSON logic | `HasJsonPreferences` trait |
| Spoiler default behaviour | Blur (configurable: skip/blur/show) |
| CW vs sensitive media | Treated separately, independent settings |
| Dedup strategy | Content similarity via `similar_text()`, ‚â•80%, within 24h, min 30 chars |
| Age cutoff scope | Per-user default, per-account override |
| Empty queue after cutoff | Naturally handled ‚Äî `useFeedQueue` fetches more; no special "nothing recent" state needed since boosted posts always pass through |
